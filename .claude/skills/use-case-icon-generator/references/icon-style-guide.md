# 使用案例圖示樣式指南

## 概觀

使用案例圖示可作為使用案例目錄表格中的視覺識別碼。 它們必須能立即以40px顯示寬度辨識，同時維持原生1024x1024解析度的品質。

## 技術規格

| 屬性 | 值 |
| --- | --- |
| 維度 | 1024 x 1024畫素 |
| 格式 | PNG （8位元RGB或RGBA） |
| 檔案大小 | 900KB — 標準值1.4MB |
| 顯示大小 | 目錄表格中的40px寬度 |
| 命名 | `icon-{kebab-case-name}.png` |
| 儲存路徑 | `/help/blueprints/industry-use-cases/assets/use-case-icons/{industry}/` |

## 視覺風格指南

### 組合
- **單一中央主旨** — 每個圖示都應該有一個清晰的視覺隱喻來代表使用案例的概念
- **居中的組合** — 主要主題應該以平衡的空白字元居中
- **沒有文字** — 文字在40px時無法辨認；完全依賴視覺隱喻
- **沒有複雜場景** — 避免使用多元素組合、包含許多物件的背景或敘述場景
- **粗體剪影** — 圖示的形狀應該可辨識，即使是一個微小的剪影

### 色彩和色調
- **乾淨的現代色盤** — 使用專業的、適合公司的色彩
- **產業凝聚力** — 相同產業內的圖示應該感覺像是屬於一起
- **高對比度** — 確保主要主旨與背景明顯不同
- **避免使用霓虹色或過度飽和的色彩** — 讓調色盤保持精緻和專業

### 樣式
- **現代與專業** — 線條乾淨，表面光潔
- **一致的呈現** — 所有圖示應該看起來都來自相同的設計系統
- 可以接受&#x200B;**3D或平面** — 但在垂直產業內是一致的
- **避免真實感效果** — 為了保持一致性，優先使用圖解或演算樣式
- **沒有品牌標誌或商標影像** — 使用一般視覺隱喻

### 小型易讀性測試
在定稿前，請在心裡將影像縮放至40px：
- 您可以識別主要主旨嗎？
- 前景和背景之間的對比是否足夠？
- 是否有任何細微的細節會變成視覺雜訊？
- 形狀是否不含顏色而清晰讀取（若為協助工具）？

## 影像產生提示範本

使用此範本作為起點，自訂每個使用案例的主旨和詳細資訊：

```
A clean, modern icon illustration of {visual metaphor} representing {use case concept}. Professional corporate design style with bold shapes and clean lines. Single centered subject on a {solid/gradient} background. High contrast, vibrant but refined color palette. No text, no fine details, no complex backgrounds. The icon must be clearly recognizable when scaled down to 40 pixels. Square format, 1024x1024 pixels.
```

### 提示自訂提示

- **請具體說明主題** — 「內含兩個彩色禮盒的鮮紅色購物車」優於「購物車」
- **指定背景處理** — 「在柔和的藍色漸層背景上」或「在帶有細微陰影的乾淨白色背景上」
- **提及照明** — 「柔和的工作室照明」或「柔和的環境光暈」有助於達成一致性
- **新增樣式修飾元** — 「極簡型」、「幾何」、「等角型」或「平面設計」以主導美學
- **如果支援則包含負面提示** — 「沒有文字、沒有浮水印、沒有邊框、沒有逼真的像片元素」
