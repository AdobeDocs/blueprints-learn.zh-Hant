# 使用案例圖示產生器

產生詳細的圖示規格和AI影像產生提示，以在藍圖存放庫中取得使用案例圖示。

## 何時該使用此技能

- 建立需要圖示的新使用案例
- 由產業使用案例建立工具技能呼叫，以產生圖示規格
- 使用者詢問關於建立、更新或取代使用案例圖示
- 使用者想要針對新的垂直產業產生一批圖示

## 步驟1：收集輸入

從使用者或通話技能收集下列資訊：

**必要：**
- **使用案例名稱** — 使用案例的可讀取名稱（例如「放棄的購物車」、「潛在客戶評分」）
- **垂直產業** — 其中之一：汽車、b2b、金融服務、醫療保健、保險、媒體娛樂、零售、電信、旅遊服務
- **簡短說明** — 1-2個描述使用案例的句子

**選擇性：**
- **偏好的視覺隱喻或主旨** — 如果使用者對於圖示應該描繪什麼有特定的想法

如果遺失任何必要的輸入，請先詢問使用者，然後再繼續。

## 步驟2：閱讀樣式指南

閱讀檔案`references/icon-style-guide.md` （相對於此技能的目錄）以載入完整的樣式指南，包括：

- 技術規格
- 視覺樣式指南
- 依產業區分的視覺隱喻範例
- 提示範本
- 現有圖示詳細目錄

使用樣式指南來確保與現有圖示一致。

## 步驟3：產生圖示規格

產生圖示規格的全部三個部分：

### A.檔案規格

從使用案例名稱和產業衍生檔案名稱和路徑：

- **檔案名稱：** `icon-{kebab-case-name}.png`
   - 將使用案例名稱轉換為大寫（小寫，空格使用連字型大小）
   - 範例：「放棄的購物車」成為`icon-abandoned-cart.png`，「交叉銷售向上銷售」成為`icon-cross-sell-upsell.png`
- **路徑：** `/help/blueprints/industry-use-cases/assets/use-case-icons/{industry}/`
- **尺寸：** 1024 x 1024畫素
- **格式：** PNG、8位元RGB或RGBA
- **目標檔案大小：** 900KB - 1.4MB

### B.影像產生提示

為AI影像產生工具建立細節、可複製提示。 提示必須：

1. **描述清楚的視覺隱喻** — 選擇能立即傳達使用案例概念的單一強大視覺隱喻。 如果使用者提供偏好的比喻，請使用它。 否則，請根據樣式指南範例和使用案例說明選取一個範本。

2. **指定藝術風格** — 包含下列風格指示：
   - 簡潔的現代圖示插圖樣式
   - 專業的企業設計美學
   - 粗體形狀和簡潔的線條
   - 光亮、演算後的光潔度（不逼真）
   - 與統一設計系統一致

3. **包含技術規格：**
   - 正方形格式，1024x1024畫素
   - 以單一為中心的主旨
   - 純色或細微的漸層背景
   - 主旨與背景的高對比度

4. **強制小型易讀性：**
   - 無任何文字或字母
   - 沒有細微的細節或複雜的圖案
   - 沒有複雜的背景或多元素場景
   - 以40px顯示清晰的粗體剪影
   - 即使沒有顏色，也能識別強大的形狀

5. **引導調色盤：**
   - 適合企業的專業色彩
   - 生動但優雅（不霓虹燈或過度飽和）
   - 協助工具的高對比度
   - 與相同垂直產業中的其他圖示結合

將提示結構為單一段落，可隨時貼到任何影像產生工具中。

### C.目錄參考程式碼片段

產生HTML程式碼片段以用於目錄表格：

```
<img src="assets/use-case-icons/{industry}/icon-{name}.png" alt="{Alt Text}" width="40">
```

其中`{Alt Text}`是人類看得懂的使用案例名稱。

## 步驟4：列出相同產業中的現有圖示

檢查樣式指南的現有圖示詳細目錄區段，並列出相同產業的所有目前圖示。 這可協助使用者：
- 產生新圖示時，請確保視覺上的一致性
- 避免複製已存在的圖示
- 以現有圖示作為樣式範例參考

## 步驟5：顯示輸出

使用以下區段清楚格式化輸出：

## 準則

- **永遠不產生實際影像** — 此技能只會產生規格和提示。 使用者必須使用外部影像產生工具。
- **每個使用案例有一個圖示** — 每個使用案例只會有一個圖示。
- **檢查重複專案** — 如果詳細目錄中已經存在相同或非常相似的圖示，請警告使用者。
- **排定易讀性的優先順序** — 如果對詳細隱喻和簡單隱喻有疑問，請務必選擇在40px下讀得很好的較簡單選項。
- **在提示中指定** — Vague提示會產生不一致的結果。 包含具體的視覺細節（例如「內含兩個彩色盒子的購物車」而非「購物概念」）。
- **儘可能避免陳詞濫調** — 嘗試尋找新鮮但仍可立即識別的視覺隱喻。 每個「智慧」使用案例的燈泡都重複出現。
