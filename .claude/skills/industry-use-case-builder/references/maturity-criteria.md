# 使用案例成熟度評估標準

## 成熟度等級

### 基礎

**徽章：** `[!BADGE Foundational]{type=Neutral}`

使用案例是&#x200B;**基礎**，當它：

- 對應至單一步驟或事件觸發的模式（事件觸發訊息、批次傳出訊息啟用）
- 具有直接的資料需求（單一事件型別、標準設定檔屬性）
- 需要最少或沒有AI/ML功能
- 使用標準通道傳送（電子郵件、簡訊、推播），無需複雜協調
- 擁有既定的實作模式，並具備清晰的最佳實務
- 需要較少的系統整合（通常是1-2個資料來源）

**典型模式：**&#x200B;事件觸發的傳訊、批次傳出訊息啟用

**範例：**&#x200B;放棄購物車復原、約會提醒、服務通知、價格下降提醒、帳單付款提醒

### 新興

**徽章：** `[!BADGE Emerging]{type=Informative}`

使用案例是&#x200B;**新興**，當它：

- 對應至多步驟或個人化模式（多步驟協調歷程、已知訪客Personalization、行為推薦、匿名訪客Personalization）
- 需要行為資料收集和分析
- 使用建議模型或已知訪客設定檔解析度
- 使用分支邏輯處理多重觸控培養序列
- 具有中等整合複雜性（2-4個資料來源）
- 可能需要對象啟動或區段型鎖定目標

**典型模式：**&#x200B;多步驟協調歷程、已知訪客網頁/應用程式Personalization、行為建議、匿名訪客網頁Personalization、B2B Audience Activation

**範例：**&#x200B;歡迎系列、藥物遵守行銷活動、個人化帳戶儀表板、潛在客戶評分、內容推薦

### 進階

**徽章：** `[!BADGE Advanced]{type=Caution}`

使用案例是&#x200B;**進階**，當它：

- 對應至決策或跨頻道協調模式（具有決策的跨頻道歷程、Offer Decisioning）
- 需要AI支援的預測或傾向分數
- 同時跨多個管道使用集中式決策邏輯
- 涉及在多個歷程點進行複雜協調與即時決策
- 具有高整合複雜性（4個以上的資料來源、即時資料摘要）
- 需要複雜的商業規則、優先順序管理和頻率控管

**典型模式：**&#x200B;具有決策的跨管道歷程，Offer Decisioning

**範例：**&#x200B;使用AI評分的流失預防、終生階段產品推薦、交叉銷售最佳化、忠誠計畫個人化、VIP專屬優惠

## 評估工作流程

1. 識別使用案例對應的使用案例模式
2. 檢查上方的「典型模式」清單，以快速比對
3. 如果模式未明確表示成熟度，請評估資料複雜性、AI/ML需求和整合範圍
4. 以推理方式向使用者呈現評量：「根據模式對應({pattern})和{key factors}，我會將此分類為{level}，因為{reasoning}。」
5. 讓使用者在繼續之前確認或覆寫
